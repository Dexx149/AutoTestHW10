# **План автоматизации тестирования сценария записи на профессию «Инженер по тестированию»**

## **1. Перечень автоматизируемых сценариев**

### **1.1. Переход на страницу профессии с главной страницы**

- Через меню "Каталог курсов":
  - Ввод названия в строку поиска
  - Выбор категории и поиск в списке курсов
- Через главную страницу:
  - Выбор категории с главной страницы и поиск в списке курсов
  - Через раздел "Популярные программы"
  - Прохождение теста, после выбор курса

### **1.2. Переход на форму записи со страницы профессии**

- Пролистывание страницы до формы записи
- Клик по кнопке "Записаться" в верхней части страницы

### **1.3. Сценарии заполнения формы записи**

- Позитивные сценарии:
  - Заполнение формы валидными данными
  - Проверка сообщений об ошибках при невалидных данных
- Негативные сценарии:
  - Пустые поля
  - Невалидный формат email
  - Невалидный номер телефона
  - Специальные символы в поле имени
  - Превышение максимальной длины полей

## **2. Перечень используемых инструментов с обоснованием выбора**

### **2.1. Язык программирования**

- Java - популярный в автоматизации, большой выбор библиотек

### **2.2. Фреймворк для автоматизации**

- Selenide - упрощает работу с Selenium, повышает стабильность тестов.

### **2.3. Система сборки**

- Maven или Gradle - обеспечивают стандартизированную структуру проекта и простоту конфигурации

### **2.4. Фреймворк для тестирования**

- JUnit 5 - современный фреймворк для автоматизированного тестирования, поддерживает параметризованные тесты

### **2.5. Дополнительные библиотеки**

- Faker - генерация тестовых данных
- Allure - создание отчетов
- Lombok - генерация кода

### **2.6. CI/CD**

- Jenkins, Github Actions, Gitlab

## **3. Перечень необходимых разрешений, данных и доступов**

### **3.1. Разрешения**

- Разрешение на автоматизацию тестирования от владельца продукта
- Доступ к тестовой среде

### **3.2. Тестовые данные**

- Набор валидных тестовых данных
- Набор невалидных данных для негативных сценариев

### **3.3. Технические доступы**

- Доступ к логам
- Доступ к CI/CD системе

## **4. Перечень и описание возможных рисков при автоматизации**

### **4.1. Технические риски**

- Изменение структуры сайта
- Медленная работа сайта

### **4.2. Организационные риски**

- Отсутствие тестовой среды
- Изменение требований
- Недостаток времени
- Отсутствие технической документации

## **5. Перечень необходимых специалистов для автоматизации**

- QA Automation Engineer (1) - разработка и поддержка автотестов
- Frontend Developer (1) - помощь с селекторами

## **6. Интервальная оценка с учётом рисков в часах**

- Настройка окружения (5-10 часов)
- Разработка автотестов (20-30 часов)
- Интеграция и доработки (10 часов)
- Резерв на риски (15-20 часов)

**Итого: 50-70 часов**
