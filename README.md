# **План автоматизации тестирования сценария записи на профессию «Инженер по тестированию»**

## **1. Перечень автоматизируемых сценариев**

### **1.1. Сценарии перехода на страницу профессии**

**S1: Переход через меню "Каталог курсов"**

Шаги:
1. Открыть главную страницу сайта
2. Кликнуть на меню "Каталог курсов"
3. Кликнуть на категорию "Все курсы"
4. Выбрать профессию "Инженер по тестированию"

Ожидаемый результат: Открыта страница профессии "Инженер по тестированию"

**S2: Переход через поиск по названию профессии**

Шаги:
1. Открыть главную страницу сайта
2. Кликнуть на меню "Каталог курсов"
3. Ввести в поле "Инженер по тестированию"
4. Кликнуть на кнопку "Найти курс"
5. Выбрать профессию из результатов поиска

Ожидаемый результат: Открыта страница профессии "Инженер по тестированию"

**S3: Переход через раздел "Популярные программы"**

Шаги:
1. Открыть главную страницу сайта
2. Пролистать до раздела "Популярные программы"
3. Выбрать профессию "Инженер по тестированию"

Ожидаемый результат: Открыта страница профессии "Инженер по тестированию"

**S4: Переход после прохождения профориентационного теста**

Шаги:
1. Открыть главную страницу сайта
2. Пролистать до раздела "Если не знаете, с чего начать"
3. Нажать кнопку "Пройти тест"
4. Пройти тест
5. Дождаться рекомендации профессии "Инженер по тестированию"
6. Выбрать рекомендованную профессию

Ожидаемый результат: Открыта страница профессии "Инженер по тестированию"

### **1.2. Сценарии перехода к форме записи**

**S5: Пролистывание страницы профессии до формы записи**

Шаги:
1. Открыть страницу профессии "Инженер по тестированию"
2. Пролистать страницу вниз до формы записи

Ожидаемый результат: Форма записи отображается на экране

**S6: Клик по кнопке "Записаться" в верхней части страницы**

Шаги:
1. Открыть страницу профессии "Инженер по тестированию"
2. Найти кнопку "Записаться" в верхней части страницы
3. Кликнуть на кнопку "Записаться"

Ожидаемый результат: Произведен скролл к форме записи, форма отображается на экране

### **1.3. Сценарии заполнения и отправки формы**

#### **Позитивные сценарии**

**S7: Успешная отправка формы с валидными данными**

Шаги:
1. Открыть форму записи на профессию
2. Заполнить поле "Имя" значением: "Мария"
3. Заполнить поле "Email" значением: "MariaKL@mail.ru"
4. Заполнить поле "Телефон" значением: "+79991234567"
5. Активировать чекбокс "Принимаю условия пользовательского соглашения"
6. Кликнуть кнопку "Получить консультацию"

Ожидаемый результат: Отображается сообщение об успешной отправке формы

#### **Негативные сценарии**

Шаги:

Ожидаемый результат: Под полем "Email" отображается сообщение об ошибке "Неверный Email"

**S10: Заполнение формы с невалидным номером телефона**

Шаги:
1. Открыть форму записи на профессию

Шаги:

2. Заполнить поле "Имя" значением: "Мария"

## **2. Перечень используемых инструментов с обоснованием выбора**

### **2.1. Язык программирования**
- Java - популярный в автоматизации, большой выбор библиотек

### **2.2. Фреймворк для автоматизации**
- Selenide - упрощает работу с Selenium, повышает стабильность тестов

### **2.3. Система сборки**
- Maven или Gradle - обеспечивают стандартизированную структуру проекта и простоту конфигурации

### **2.4. Фреймворк для тестирования**
- JUnit 5 - современный фреймворк для автоматизированного тестирования, поддерживает параметризованные тесты

### **2.5. Дополнительные библиотеки**
- Faker - генерация тестовых данных
- Allure - создание отчетов
- Lombok - генерация кода

### **2.6. CI/CD**
- Jenkins, Github Actions, Gitlab

## **3. Перечень необходимых разрешений, данных и доступов**

### **3.1. Разрешения**
- Разрешение на автоматизацию тестирования от владельца продукта
- Доступ к тестовой среде

### **3.2. Тестовые данные**
- Набор валидных тестовых данных

### **3.3. Технические доступы**
- Доступ к логам
- Доступ к CI/CD системе

## **4. Перечень и описание возможных рисков при автоматизации**

### **4.1. Технические риски**
- Изменение структуры сайта
- Медленная работа сайта

### **4.2. Организационные риски**
- Отсутствие тестовой среды
- Изменение требований
- Недостаток времени
- Отсутствие технической документации

- QA Automation Engineer (1) - разработка и поддержка автотестов

- Настройка окружения (5-10 часов)

- Интеграция и доработки (10 часов)
